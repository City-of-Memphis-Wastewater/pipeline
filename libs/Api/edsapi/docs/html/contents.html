SDK contents

The EDS API SDK has the following directory structure:

  • backend

    Contains precompiled Backend libraries libraries which need to be shipped with applications that use the EDS API.

    The libraries are located in several subdirectories - one for each supported EDS version.

    When the API is integrated into an application, the .dll or .so libraries (depending on the platform) need to be copied to a location where the application can find them. The list of recommended paths for the libraries is given in the Search paths for backend libraries page.

    Note that applications which do not connect to a specific EDS version (e.g. 9.1) do not have to ship with that particular version of backend libraries.

  • docs

    The documentation for EDS API.

  • examples

    Examples of console applications which use EDS API to access live point data (LiveClient.cpp) and EDS archives (ArchClient.cpp).

    The Compiling examples with Visual Studio guide explains how to build these examples on Windows.

  • include

    Contains C++ headers for EDS API client classes.

    Projects which need to call EDS API functions typically have to ensure that these files are available in the default search path for C++ headers. Please refer to the Configuring EDS API headers section for more information on the subject.

  • src

    Contains C++ source code for the Frontend library.

    When integrating the EDS API into an application, the instructions in Building EDS API clients page should be followed to prepare the frontend library.
