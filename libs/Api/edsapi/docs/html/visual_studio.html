Compiling examples with Visual Studio

Following guide illustrates how to compile EDS API live data example with Visual Studio 2017. It also assumes that 32 bit API is used and example is compiled as 32 bit command line application with precompiled headers turned off.

  1. Create a new command line project in Visual Studio.

  2. Import EDS Frontend layer source files to the project (both cpp and header files. By default the files are located in C:\Program Files (x86)\EDS92\Api\edsapi\src).

    Project's file list should look similar to the one on screenshot below:

  3. Add C:\Program Files (x86)\EDS92\Api\edsapi\include to project's "Additional Include Directories".
  1. Build the project.
  2. Replace contents of main project file with contents of C:\Program Files (x86)\EDS92\Api\edsapi\examples\LiveClient.cpp.

  3. Change point names used in example after comment

    // Points have to be subscribed to receive value updates from server

    and

    // Points have to be originated to send value changes to server

    to names used in your EDS system (point names can be copied from Terminal Point Info window).

  1. Copy files from C:\Program Files (x86)\EDS92\Api\edsapi\backend\9.2 to a folder where projectâ€™s compiled executable was placed.

  2. Run the example using following arguments:

    <app_name>.exe <eds_version> <instance_name> <eds_server_ip_addr> <eds_server_port>

    • <eds_version> - use 9.2
    • <instance_name> - used for connection identification, can be anything e.g "example"
    • <eds_server_ip_addr> - EDS server IP address
    • <eds_server_port> - UDP port used by server, by default 43000

    e.g

    EDS_API_Live_Data_Example.exe 9.2 live_data_example localhost 43000

    Application should read and write selected points for 60 s.
