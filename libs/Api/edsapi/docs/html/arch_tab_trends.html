Executing tabular trends

The archive client from EDS API can also execute tabular trends on Archive Server.

A tabular trend consists of a set of function calls that are applied to archive samples of specified points.
The functions are executed in steps, for consecutive periods of time. Every step results in a separate row of data (with one column per function call/point).

In general almost all functions which can be applied in report calls are also available for tabular trends.
For a detailed list of functions and their parameters, please refer to the EDS User Manual.

Defining a tabular trend

Before a tabular trend can be executed on Archive Server, each function call/point must be registered in eds::arch::ArchClient. Note that the client should be already initialized.

The example below constructs two function calls - one for the VALUE function, the other for AVG.

The maxValueSid and avgSid variables used in this example should contain the SID identifiers of queried points. SIDs can be obtained from an IESS identifiers using eds::live::LiveClient. Please see the Process point identifiers page for more information.

try
{
time_t now = time(nullptr);
// Pointers are wrapped in unique_ptr to automatically release objects
// in case of exception.
// Create a VALUE trend with preference for shade values over archives.
std::unique_ptr<TabularTrend> valueTabTrend(client->getTabularTrend(valueSid, "VALUE"));
client->addTabularTrend(valueTabTrend.get(), ShadeMode_PreferShade);
// Create an AVG trend.
std::unique_ptr<TabularTrend> avgTabTrend(client->getTabularTrend(avgSid, "AVG"));
client->addTabularTrend(avgTabTrend.get());
// Tabular trend will be executed here in the next step.
}
catch (const ArchClientError& exc)
{
// Handle the error (display an error message, etc).
printf(
"Couldn't create definition of a tabular trend (%s)\n",
exc.what());
}

Executing a tabular trend

The tabular trends defined in the previous step can be now executed. Call the eds::arch::ArchClient::executeTabularTrends method with a reference time, step length, and total time range of the trend. The call will block until the tabular trend is ready.

try
{
// Execute both trends for last 5 hours with 1 hour step size.
// This function will block until trend results are ready.
client->executeTabularTrends(now - 5 * 3600, 5 * 3600, 3600);
}
catch (const ArchClientError& exc)
{
// Handle the error (display an error message, etc).
printf(
"Couldn't execute a tabular trend (%s)\n",
exc.what());
}

Fetching values from a tabular trend

To access the result of a tabular trend execution, use the eds::arch::ArchClient::fetchTabularTrendRow method.

The function should be called in a loop until it returns false. At each step it will return a single row of values (which corresponds to data from one step of calculations).

try
{
TabularTrendRow row;
while (client->fetchTabularTrendRow(&row))
{
for (size_t i = 0; i < row.size(); ++i)
printf("%12.6f%c\t", row[i].value, row[i].quality);
printf("\n");
}
}
catch (const ArchClientError& exc)
{
// Handle the error (display an error message, etc).
printf(
"Couldn't fetch values from a tabular trend (%s)\n",
exc.what());
}
