Detailed Description

Live data client.

Objects of this class allow retrieval and updating of live process point parameters.

Only one live backend can be loaded at any time. Do not create multiple LiveClient objects with different values of version parameter. Multiple LiveClient objects with the same version are allowed.

Most methods of this class may throw exceptions:

  • LiveClientError - backend library returned an error. Use LiveClientError::errorCode() to get more information on error type.
  • UninitializedClientError - initializeAsAgent() or initializeAsUser() method was not called or LiveClient was disconnected using shut() method.
  • UnsupportedFunctionError - called method is not supported by currently loaded backend. This usually happens when feature was not available in that particular EDS version.

Methods of this class are not thread safe and can be called from one thread only.

Examples
ArchClient.cpp, and LiveClient.cpp.

◆ setInput()

void eds::live::LiveClient::setInput ( int lid)

Registers LiveClient as receiver of dynamic values for given point.

Points must be marked as input to receive updates of dynamic fields (value, quality, status) from server. Note that static point fields (IESS, DESC, AUX) can be read without calling this method.

LiveClient internally uses a counter which is incremented by every invocation of setInput(). This means that every call to setInput() must be paired with a call to unsetInput() on the same point.

LiveClient must be in AccessMode_Read or AccessMode_ReadWrite access mode to receive point values.

It is illegal to mark a point as both input and output.

Examples
LiveClient.cpp.

void eds::live::LiveClient::setOutput ( int lid)

Registers LiveClient as a source of values for given point.

◆ synchronizeInput()

void eds::live::LiveClient::synchronizeInput ( )

Receives updated point values from server.

This function should be called periodically (typically once per second) in LiveClient's read mode. It downloads updated values for dynamic fields (value, quality, status) of all subscribed points (see setInput()).

If this method throws a LiveClientError exception with any of the following error codes:

  • LiveErrorCode_NotLoggedIn
  • LiveErrorCode_NotSynchronized

then it is possible to continue using this LiveClient instance. These errors are temporary and may be resolved by calling synchronizeInput() method again (possibly multiple times).

Examples
LiveClient.cpp.

◆ staticInfoChanged()

bool eds::live::LiveClient::staticInfoChanged ( )

Checks if static part of point list has changed.

This method returns true if the static part of point list has changed on the server. This happens when a value of a static point field was modified, or when a point was added / deleted.

Parameters

[in]value- value containing ST bits to be written (see PointStatus.h)

Since
EDS 9.2

Before calling this function, written point must be set for output (see setOutput() function). Updated ST values will be sent to server during the next call to synchronizeOutput(). Bits of the ST field are defined in the header file PointStatus.h

Important notes:

  • bit eds::StatusBit::ST_OFFSCAN cannot be set by this API.
  • bits carrying:
    • Alarm Pririty (mask 0x00070000, bits 16, 17, 18)
    • Alarm Number (mask 0x00700000, bits 20, 21, 22) have values decremented by 1 compared to the Alarm Priority or Alarm Number they are encoding. For example, the following call: 
        writeST(<lid>, 0x00130088, 0x007700FF);
      will set high alarm with attributes:
      • Alarm Priority = 4
      • Alarm Number = 2 on Alarm List in EDS Terminal.
