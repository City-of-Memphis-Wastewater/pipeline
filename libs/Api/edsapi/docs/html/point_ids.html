Process point identifiers

EDS assigns two numeric identifiers to every process point:

  • a local identifier (abbreviated as LID)
    These identifiers are assigned on the client side (they may be different for every client machine connecting to the same EDS server). They are always in range <0; eds::live::LiveClient::highestLID()>.
    However, not all identifiers in that range must be valid. Some of them may refer to deleted points. The eds::live::LiveClient::isPointAlive() method can be used to determine whether a point is deleted.
    Negative identifiers are invalid and are often returned by EDS methods when a point cannot be found.
  • a server-side identifier (abbreviated as SID)
    These identifiers are assigned by the server and are consistent on all clients connected to that server.
    Valid server identifier are always higher than 0.

These types of identifiers may not be used interchangeably. All EDS API methods clearly document, which point identifier type is expected (both in method header and in this API reference).
In general eds::live::LiveClient methods expect local identifiers and eds::arch::ArchClient methods expect server-side identifiers.

To convert a local identifier to a server-side identifier, please use the eds::live::LiveClient::pointSID() method.

Each EDS Process point also has two string identifiers:

  • IESS - unique for the whole deployment of EDS
    Call eds::live::LiveClient::findByIESS() to convert it to a LID.
  • IDCS - unique within the point's data source (ZD)
    Call eds::live::LiveClient::findByIDCS() to convert it to a LID.
