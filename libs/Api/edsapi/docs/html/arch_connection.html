Connecting to EDS Archive Server

  1. To access data on EDS Archive Server, applications must first create an instance of eds::arch::ArchClient.

    The class is declared in the ArchClient.h header, which should be available to the application, provided that the steps described in the Configuring EDS API headers section were completed.

    In the application code, add the following #include directive:

    #include <ArchClient.h>
    ArchClient.h
    Client for archive data.
  2. Most of the classes are located in the eds and eds::arch namespaces. You may choose to make their contents available by default:
    // Some common types are defined in eds namespace
    using namespace eds;
    // ArchClient is in eds::arch namespace
    using namespace eds::arch;

  3. Create an instance of the eds::arch::ArchClient class.

    std::unique_ptr<ArchClient> client;
    try
    {
    client = std::unique_ptr<ArchClient>(new ArchClient(version));
    }
    catch (const BackendNotFoundError& exc)
    {
    // Couldn't load backend. Requested version might be incorrect,
    // or the 'edsapi_arch_*' library might be missing.
    }
    eds::BackendNotFoundError
    Backend not found exception class.
    Definition Errors.h:39
    eds::arch::ArchClient
    Archive data client.
    Definition ArchClient.h:118

    The version argument should match the version of the Archive Server which will be used. It is specified as a string with major.minor format (e.g. "9.2").

    Note that the code is wrapped in a try/catch statement. The constructor may throw an eds::BackendNotFoundError exception if the required backend library could not be found.

    To prevent these errors, make sure that backend libraries are correctly deployed with the application. For more information on recommended locations for the backend files, see Search paths for backend libraries.

    The code in this example uses std::unique_ptr to ensure that the client is correctly released once the object goes out of scope. If C++11 is unavailable, it is recommended to use an alternative type (e.g std::auto_ptr) to manage lifetime of pointers returned from EDS API.
    eds::arch::ArchClient uses exceptions to report errors, so applications which call its methods should ensure that their code is exception-safe.

  4. Initialize a connection to Archive Server

    The eds::arch::ArchClient instance must be initialized with the address of target Archive Server.

    try
    {
    client->init("0.0.0.0", // bind to all local interfaces
    0, // bind to any local port
    archive_server_host,
    archive_server_port,
    archive_server_lport_range);
    }
    catch (const ArchClientError& exc)
    {
    // Handle the error (display an error message, etc).
    printf(
    "Couldn't initialize arch client connection object (%s)\n",
    exc.what());
    }
    eds::arch::ArchClientError
    Thrown when backend library returns an error.
    Definition ArchClient.h:522
  5. The connection will be automatically closed by the destructor of eds::arch::ArchClient object. Alternatively, the eds::arch::ArchClient::shut method can be used to manually close the connection.
