Writing shade values to archive

Shade values are auxiliary data which can override contents of actual EDS archive.

A single shade value consists of:

  • a point identifier (SID),
  • a start time,
  • an end time,
  • a point value,
  • quality.

Any number of shades can be written to EDS database, as long as their time ranges don't overlap for any given point.

To improve performance shades are written in batches.

First, each shade value is added to a batch using the eds::arch::ArchClient::addShadeValue method.

Then all shades are committed to EDS database by calling eds::arch::ArchClient::writeShades. The function will block until the operation is completed.

try
{
time_t now = time(0);
// Write 10.0G for last 5 hours.
client->addShadeValue(sid1,
now - 5 * 3600,
now,
10.0,
Quality_Good);
// Write 20.0G for range <-5 hours; -2 hours>.
client->addShadeValue(sid2,
now - 5 * 3600,
now - 2 * 3600 - 1,
20.0,
Quality_Good);
// Write 40.0F for range <-2 hours; +5 hours>.
client->addShadeValue(sid2,
now - 2 * 3600,
now + 5 * 3600,
40.0,
Quality_Fair);
// This function will block until all shades are written.
client->writeShades();
}
catch (const ArchClientError& exc)
{
// Handle the error (display an error message, etc).
printf(
"Couldn't write shade values (%s)\n",
exc.what());
}
