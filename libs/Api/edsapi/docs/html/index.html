Overview

EDS API is a set of C++ header files and libraries which allows to connect to EDS servers from external software.

Libraries provided in this SDK support Win32 and Linux and work with all versions of EDS from 9.1 upwards. See the Requirements page for detailed information on supported platforms.

In order to support a range of EDS versions without having to constantly recompile and link software, EDS API has been split into two layers:

  • Frontend library - classes which load backend libraries. They are statically linked with the application which uses the API.
  • Backend libraries - libraries which implement the network protocol of EDS Online and Archive Server. They are provided in binary form as .dll or .so files. Each major EDS version requires a separate set of backend libraries.

Please refer to the Architecture page for more information on the topic.

The overall structure of the EDS API SDK is described in the SDK contents page.

For a step-by-step introduction to the API, visit the Tutorial.

The SDK also provides sample console applications which use the API to obtain data from EDS servers:

  • LiveClient.cpp - downloads live point data from EDS Online Server,
  • ArchClient.cpp - demonstrates how to execute a report function, download a tabular trend and write shade values to EDS archive.

For instructions on how to build these examples with Visual Studio, please visit the Compiling examples with Visual Studio page.

For detailed description of all classes and methods, consult the API reference.
